How to Integrate Esewa Epay-V2 in Laravel application Using Livewire

 

 

Hello In this article I am going to integrate Esewa epay-v2 in Laravel application using livewire v3 .

Lets start our project by creating new project and installing livewire

laravel new esewatest

composer require livewire/livewire

Now we need to create a app layout for out application.

php artisan livewire:layout
//this will create a new app.blade.php file in components/layouts directory.

After this lets create a livewire components :

php artisan make:livewire CheckoutPage

Now will add 2 functions . getPaymentDetails() and makePayment() .

public function getPaymentDetails(){
$tuid=now()->timestamp;
$message="total_amount=110,transaction_uuid=$tuid,product_code=EPAYTEST";
$s = hash_hmac('sha256', $message, '8gBm/:&EnhH.1/q', true);
$signature= base64_encode($s);
$dataArray = [
"amount" => "100",
"failure_url" => route('esewa.failure'),
"product_delivery_charge" => "0",
"product_service_charge" => "0",
"product_code" => "EPAYTEST",
"signature" => $signature,
"signed_field_names" => "total_amount,transaction_uuid,product_code",
"success_url" => route('esewa.success'),
"tax_amount" => "10",
"total_amount" => "110",
"transaction_uuid" => $tuid
];
return $dataArray;
}

public function makePayment(){
$data=$this->getPaymentDetails();
$this->dispatch("redirectToEsewaPage",$data);
}

In getPaymentDetails() , 8gBm/:&EnhH.1/q is a secret key for testing environment. In Production this key is replaced by The Merchant key. Similary $message is string which contains the totalamount,transactionid, and product_code of your application.transaction id can be generated using php scripts.

Now we need two more routes for handling payment success or failure.for that I have created a TestController file which looks like:

class TestController extends Controller
{
public function esewaSuccess(Request $request){
$decodedString = base64_decode($request->data);
$data=json_decode($decodedString,true);
if(array_key_exists("status",$data)){
$status = $data["status"];
if($status==="COMPLETE"){
dd("It is completed");
}
}
dd($request->all());
}
public function esewaError(Request $request){
dd($request->all());
}
}

Here request->data represents the data provided by the esewa while redirecting to our server after the payment procedure. It is is base64 encoded form . first we need to decode this then decode again as a php array.

Until now , We dont have used any js in our application .Unfortunately Esewa does not provide a way to directly redirect to another page using backend only. So we must send our payment data to the client then submit a form using js to post our form . which will redirect us to the esewa server.

So To Achieve this we need a listen for a event in our blade file which will create a form ,append it in the body then submit it.

In makePayment() method, it is dispatching a event namely redirectToEsewaPage. Which Being Listened In The Blade file Which Looks Like:

<div>
<button wire:click="makePayment" >Submit</button>
</div>

@script
<script>
$wire.on("redirectToEsewaPage",(data)=>{
data=data[0];
var path="esewa_api";
var form=document.createElement("form")
form.setAttribute("method","POST")
form.setAttribute("action",path)

var formData = new FormData();
for (var key in data) {
var hiddenField=document.createElement("input")
hiddenField.setAttribute("type","hidden")
hiddenField.setAttribute("name",key)
hiddenField.setAttribute("value",data[key])
form.append(hiddenField)
}
document.body.appendChild(form)
form.submit()
})
</script>
@endscript

Here . When a user clicks a submit button It triggers makePayment function ,Which Creates payment details and passes this data to the redirectToEsewaPage listener in a client side. Then the js will create a form ,crate input filed for each keys and set default values then submit this.

Now it will redirect to esewa page , then performs payment procedures.
